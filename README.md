## go_weather
もっと低レイヤーな、シンプルな言語やってみたいと思ったのでGoを触ってみたいと思います。

### 目的
1. Goらしさってなに？みたいなのが知りたい
2. エラーハンドリングの理解
3. 標準ライブラリを使って、Goの力を見てみたい

### どんなアプリ？
複数都市の天気を一瞬で取得できるアプリ。


### 技術スタック
- Language: Go 1.21+

- API: Open-Meteo

- Concepts:

  - Goroutine & sync.WaitGroup (並行処理)

  - Struct Tags (JSONパース)

  - Interface & Dependency Injection (テスト用)

### 試したいこと
- [ ] 10都市にリクエストを送ったときと、1都市にリクエストを送ったときのパフォーマンスの比較
- [ ] エラーの握りつぶしをして、実行時にどんな挙動になるか観察する
- [ ] `go test`やってみたい

### ロードマップ
1. Hello Go - 環境構築と最小構成の実行
2. Single Fetch - 特定の都市の天気をAPIから取得
3. Concurrent Fetch - Goroutineを導入し、複数都市を同時並行で取得
4. Robustness - エラーハンドリングの明示化とタイムアウト処理の実装
5. Testing - Go標準の testing パッケージによる単体テストの実装

※ フロントなに使うか悩んでるので、まずはCLIで動かすことをゴールにしたい


### ディレクトリ構成イメージ
```
go_weather/
├── main.go       # エントリポイント（ここから実行）
├── weather.go    # 天気取得のロジック（Issue 2〜4で実装）
├── weather_test.go # テストコード（Issue 5で実装）
└── go.mod        # Goのモジュール管理（Issue 1で作成）
```
